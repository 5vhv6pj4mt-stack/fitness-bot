# Bug Report — Fitness Mini App
_Дата: 2026-06-11_

---

## 🔴 [HIGH] Кнопка "Записать подход" намертво блокируется при ошибке сети

**Раздел:** Зависания / Потеря данных  
**Файл:** `webapp/src/pages/Workout.jsx:134–146`

**Сценарий:** пользователь нажимает "Записать подход", в этот момент пропадает интернет или сервер возвращает 500.

**Что сейчас:** `submit()` вызывает `setLoading(true)` до `await onLog(...)`, но если `onLog` бросает исключение — `setLoading(false)` никогда не выполняется. Кнопка остаётся `disabled` навсегда до перезагрузки.

**Фикс:** обернуть в `try/finally`:
```js
try {
  setLoading(true)
  await onLog(...)
} finally {
  setLoading(false)
}
```

## 🔴 [HIGH] Упражнения с весом тела отсутствуют в прогрессе и тоннаже

**Раздел:** Логика тренировок  
**Файл:** `database/db.py:229, 242, 605, 929, 997, 1011, 1024, 1036`

**Сценарий:** пользователь делает подтягивания, пресс, вис — с `weight = 0`.

**Что сейчас:** во всех 8 SQL-запросах стоит `AND ws.actual_weight > 0`. Все подходы с весом 0 не считаются в тоннаже, не показываются в истории, не учитываются в недельном отчёте.

**Затронутые упражнения Кирилла:**
- Подтягивания широкие
- Подтягивания обратным хватом
- Пресс
- Вис на перекладине

**Фикс:** убрать фильтр `> 0` или заменить на `>= 0`.

## 🟡 [MEDIUM] Анализ тренировки молча исчезает при ошибке API

**Раздел:** Зависания / Краши  
**Файл:** `webapp/src/pages/Workout.jsx:254–256`

**Сценарий:** тренировка завершена, Claude API возвращает ошибку или таймаут при генерации анализа.

**Что сейчас:** `catch` вызывает `setAnalysisLoading(false)` но не устанавливает сообщение об ошибке. Блок анализа просто исчезает без объяснений.

**Фикс:** добавить `setAnalysis('Анализ недоступен, попробуй позже')` в catch.

## 🟡 [MEDIUM / ТРЕБУЕТ ПРОВЕРКИ] Нет валидации диапазона КБЖУ после AI-парсинга

**Раздел:** Логика питания  
**Файл:** `services/ai_service.py:96–104` + endpoint log_food в `webapp_api.py`

**Сценарий:** голосовой ввод нечёткий, Claude неверно распознаёт и возвращает 4500 ккал за "яблоко".

**Что сейчас:** `_extract_json()` конвертирует число без проверки границ. Запись уходит в БД как есть.

**Фикс:** cap на бэкенде — калории 1–3000 за приём, БЖУ 0–300г.
