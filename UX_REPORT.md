# UX-аудит — Fitness Mini App
_Дата: 2026-06-11_

---

## ⚠️ [НЕТ ПОДСКАЗКИ] RPE — шкала без объяснения

**Экран:** Тренировка
**Элемент:** таблетки 6.5 / 7 / 7.5 / 8 / 8.5 / 9 / 9.5
**Что сейчас:** подпись "RPE — усилие" и ряд кнопок с числами. Пользователь не знает что такое 6.5, почему начинается не с 1, в какую сторону лучше.
**Что должно быть:** подсказка рядом с заголовком:
«RPE — насколько тяжело. 6.5 = легко, 9.5 = предел»
**Где:** Workout.jsx:186–187

---

## ⚠️ [НЕТ ПОДТВЕРЖДЕНИЯ] Тренировка завершается без подтверждения

**Экран:** Тренировка
**Элемент:** кнопка "🏁 Завершить тренировку"
**Что сейчас:** нажал — тренировка сразу закрывается, отмены нет. Случайное нажатие не поправить.
**Что должно быть:** Telegram.WebApp.showConfirm('Завершить тренировку?')
**Где:** Workout.jsx:660–666

---

## ⚠️ [НЕОЖИДАННОЕ ПОВЕДЕНИЕ] Технические ошибки в интерфейсе

**Экран:** Питание, Профиль
**Элемент:** toast-уведомления и alert при ошибках
**Что сейчас:** пользователь может увидеть:
- "Ошибка: Failed to fetch"
- "Ошибка: Network request failed"
- "Ошибка: Empty initData"
Плюс одно место — нативный alert() с техническим текстом при записи фото.
**Что должно быть:** "Не удалось сохранить — проверь соединение"
**Где:** Nutrition.jsx:148, 313, 322 и Profile.jsx:131

---

## ⚠️ [НЕТ СОСТОЯНИЯ] Ошибка загрузки без кнопки повтора

**Экран:** Главный
**Элемент:** весь экран при ошибке API
**Что сейчас:** красный текст "Ошибка загрузки данных" — и всё. Нет кнопки "Попробовать снова".
**Что должно быть:** кнопка повтора + понятный текст
**Где:** Dashboard.jsx:76–78

---

## ⚠️ [НЕТ ПРЕДУПРЕЖДЕНИЯ] КБЖУ превышает норму без сигнала

**Экран:** Питание / Главный
**Элемент:** кольцо калорий и прогресс-бары КБЖУ
**Что сейчас:** при превышении нормы кольцо просто заполнено, числа уходят за 100% — никакого визуального сигнала.
**Что должно быть:** кольцо краснеет или появляется подпись "Норма превышена" при >110%
**Где:** Dashboard.jsx (CalorieRing, строки 14–60), Nutrition.jsx
