# Fitness Mini App — Product Document

## Что это

Telegram Mini App для трекинга тренировок и питания.
Открывается прямо из бота @stat_sila_bot — без регистрации, без паролей, без отдельного аккаунта.

---

## Анализ референса (Android-приложение)

### Что есть в референсе

| Раздел | Содержимое |
|--------|-----------|
| Рабочий стол | Карточка программы, список тренировок, календарь, 3D-тело, график веса, диаграммы нагрузки |
| Тренировки | 33 тренировки подряд пронумерованные, заблокированные замком |
| Диета | КБЖУ (не показано детально) |
| Справочник | Упражнения, спортпит, таблица калорийности, фармакология, энциклопедия |
| Ещё | Подписка, сообщество, тренерский штаб, аналитика, замеры, настройки |

### Слабые места референса

**UX и структура**
- Дашборд перегружен — 8+ виджетов, всё в один длинный скролл, важное теряется
- Список тренировок — 33 штуки пронумерованных, нет понятия недели/периодизации. Пользователь не понимает структуру
- 3D-тело красивое, но показывает не реальную нагрузку из логов, а условную окраску
- Календарь просто отмечает дни, никакой аналитики

**Функционал**
- Нет AI — программа статичная, не адаптируется под результаты
- Нет RPE — нет измерения интенсивности, только факт выполнения
- Нет прогрессии весов — приложение не знает что ты жал прошлый раз и не подсказывает рабочий вес
- Нет голосового ввода еды
- Нет анализа тренировки

**Монетизация / барьеры**
- Спортпит — только по подписке
- Социальные функции — только после регистрации отдельного аккаунта
- Фармакология прямо в приложении (стероиды, пептиды, инъекционные) — сомнительный контент

**Дизайн**
- Старомодный — таблицы, базовые карточки, бирюзово-серая палитра без характера
- Нет тёмной темы (Telegram по умолчанию тёмный)
- Замки на следующих тренировках — психологически давят, мотивации нет

---

## Наши преимущества

- Авторизация через Telegram — нулевой барьер входа
- AI (Claude) уже работает: генерация программы, разбор питания, анализ тренировки
- Голосовой ввод еды через Whisper
- Все данные уже есть в SQLite — история подходов, тоннаж, КБЖУ
- Push-уведомления уже работают через бота
- Нет подписки — личный инструмент

### 4. Прогресс / Статистика

**Цель:** увидеть рост за месяц/квартал

Вкладки:
- **Тоннаж** — линейный график по неделям
- **Веса** — прогресс по конкретному упражнению (выбор из списка)
- **Питание** — среднее КБЖУ за неделю
- **Тело** — динамика веса

**Улучшение vs референс:** реальная 3D-нагрузка на мышцы на основе логов (какие группы тренировались за последние 28 дней и с каким объёмом)

## Технический стек

| Слой | Технология |
|------|-----------|
| Frontend | React 18 + TypeScript |
| UI | shadcn/ui + Tailwind CSS |
| Telegram | @twa-dev/sdk (Telegram WebApp) |
| Тема | Автоматически подхватывает тему Telegram (тёмная/светлая) |
| Backend | FastAPI (новый сервис) или расширяем существующий бот |
| Auth | Telegram initData — никаких паролей |
| DB | Тот же SQLite, те же таблицы |
| Хостинг | Тот же VPS, nginx проксирует /miniapp/ |

## Порядок разработки (MVP → full)

### MVP (v1)
1. Backend API — эндпоинты для чтения данных из SQLite
2. Dashboard — питание + тренировка сегодня
3. Активная тренировка — логирование подходов (заменяет бот-интерфейс)
4. Питание — добавить приём пищи

### v2
5. Прогресс — тоннаж, график весов
6. Программа — просмотр цикла и упражнений
7. Голосовой ввод еды прямо из Mini App

### v3
8. Мышечный баланс — реальная нагрузка по группам
9. Рекомендации веса на основе истории + RPE
10. Офлайн-режим (PWA, Service Worker)

## Ключевые дизайн-решения

1. **Тёмная тема по умолчанию** — большинство пользователей Telegram используют тёмную
2. **Крупные кнопки** — интерфейс работает потными руками в зале
3. **Минимум текста** — иконки + числа, без абзацев
4. **Один экран = одна задача** — не пытаемся поместить всё на один скролл
5. **Хаптика** — вибрация при записи подхода (Telegram WebApp.HapticFeedback)
6. **Нет регистрации** — `Telegram.WebApp.initDataUnsafe.user` даёт всё необходимое
