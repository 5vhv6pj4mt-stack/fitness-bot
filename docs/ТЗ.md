# Техническое задание — Fitness Bot (@stat_sila_bot)

**Версия:** 1.0  
**Дата:** 2026-05-30  
**Статус:** Реализован (ретроспективное ТЗ)

---

## 1. Назначение системы

Telegram-бот для персонального трекинга силовых тренировок и питания с AI-анализом на базе LLM. Целевая аудитория — атлеты любительского и полупрофессионального уровня, занимающиеся в тренажёрном зале.

---

## 2. Цели и задачи

| Цель | Задача |
|------|--------|
| Замена бумажного дневника | Цифровая запись подходов в формате `вес×повторения RPE` |
| Контроль питания | Распознавание еды через текст или голос, расчёт КБЖУ |
| Прогрессия нагрузки | Хранение истории тренировок, сравнение с предыдущей сессией |
| Обучение технике | AI-генерация инструкций по технике + иллюстрации |
| Мотивация | Напоминания о приёмах пищи, сводка дня, статистика |

---

## 3. Пользователи системы

**Основной пользователь:** атлет, занимающийся в тренажёрном зале 3–5 раз в неделю.

Характеристики:
- Использует Telegram как основной мессенджер
- Уровень опыта: начинающий / средний / продвинутый
- Цели: набор массы / похудение / поддержание формы

---

## 4. Функциональные требования

### 4.1 Онбординг
- [ ] Сбор профиля: имя, возраст, рост, вес, цель, опыт, дни в неделю, оборудование, травмы
- [ ] AI-генерация персональной программы тренировок (Groq API)
- [ ] Расчёт целей по КБЖУ на основе профиля

### 4.2 Тренировки
- [ ] Отображение текущего дня программы (тип недели + тип дня)
- [ ] Пошаговое логирование подходов в FSM: название упражнения → `вес×повторения RPEn`
- [ ] Пропуск подхода
- [ ] Таймер отдыха с обратным отсчётом и уведомлением за 10 секунд до конца
- [ ] Возобновление незавершённой тренировки
- [ ] Завершение тренировки: расчёт тоннажа и среднего RPE
- [ ] AI-анализ тренировки (сравнение с предыдущей сессией того же типа)
- [ ] Просмотр плана тренировки и прогресса

### 4.3 Питание
- [ ] Запись приёма пищи текстом: `200г куриной грудки, 150г риса`
- [ ] Запись приёма пищи голосовым сообщением (транскрипция через Groq Whisper)
- [ ] AI-парсинг еды: название → калории, белки, углеводы, жиры
- [ ] Прогресс-бары КБЖУ относительно дневной нормы
- [ ] Подробный список приёмов пищи за день с временем и нутриентами
- [ ] AI-советы по питанию на основе текущего рациона
- [ ] Автоматические напоминания о следующем приёме пищи через 2 часа

### 4.4 Статистика
- [ ] Сводка дня: питание + тренировка
- [ ] Статистика тоннажа за последние 30 дней
- [ ] История питания по дням

### 4.5 Настройки
- [ ] Обновление веса командой `вес 75` или через меню
- [ ] Обновление целей по калориям
- [ ] Пересборка программы через `/start`

### 4.6 Энциклопедия упражнений
- [ ] Каталог 34 упражнений по 6 группам мышц
- [ ] AI-техника выполнения с биомеханическими деталями
- [ ] Иллюстрации из открытой базы wger.de
- [ ] Кэширование техники в памяти

## 5. Нефункциональные требования

| Параметр | Требование |
|----------|-----------|
| Доступность | 24/7, systemd Restart=always |
| Платформа | Telegram Bot API (Long Polling) |
| AI-ответ | < 10 сек (Groq llama-3.3-70b-versatile) |
| Голос | < 15 сек на транскрипцию + парсинг |
| Масштаб | Одиночный пользователь / малая группа |
| Хранение | SQLite (нет требований к репликации) |
| Язык ответов | Русский |
| Безопасность | Токены в .env, нет публичного API |

## 6. Сценарии использования (User Stories)

**US-01. Записать тренировку**
> Как атлет, я хочу записать подходы жима лёжа с весами и RPE, чтобы отслеживать прогрессию нагрузки.

**US-02. Записать еду голосом**
> Как пользователь, я хочу надиктовать что съел на завтрак, чтобы бот автоматически посчитал КБЖУ без ввода с клавиатуры.

**US-03. Получить технику упражнения**
> Как атлет, я хочу посмотреть технику становой тяги с иллюстрацией, чтобы избежать ошибок.

**US-04. Получить сводку дня**
> Как пользователь, я хочу видеть в одном сообщении статус питания и тренировки за сегодня.

**US-05. Возобновить тренировку**
> Как атлет, я хочу продолжить незавершённую тренировку если случайно закрыл бот.

## 8. Ограничения и допущения

- Бот работает в режиме Long Polling (не Webhook)
- Один экземпляр бота на сервере (systemd unit)
- SQLite достаточен при аудитории до ~100 активных пользователей
- Groq API — бесплатный tier с ограничением по RPM
- Изображения упражнений зависят от внешнего сервиса wger.de
