# Сессия

**ВАЖНО:** В начале каждой сессии автоматически вызывай скилл `handoff` (без команды пользователя), чтобы подхватить контекст предыдущей работы. Делай это до любого другого ответа.

## AgentMemory — обязательно писать во время работы

Используй инструмент `mcp__agentmemory__memory_save` РЕГУЛЯРНО в ходе сессии:
- После каждого значимого изменения файла — записывай что изменил и зачем
- После решения проблемы/ошибки — записывай причину и решение
- В конце сессии — записывай итоги

Формат вызова:
```
mcp__agentmemory__memory_save({
  "observation": "Что именно сделано и почему",
  "type": "decision" | "file_change" | "error_fix" | "summary"
})
```

Это позволяет вспомнить детали в следующей сессии через `mcp__agentmemory__memory_recall`.

**После каждого вызова memory_save** — обязательно писать в чат строку:
`💾 Сохранено в память: "[краткое описание что сохранено]"`

# Fitness Bot — @stat_sila_bot

Telegram-бот для трекинга тренировок и питания с AI-анализом.

**Путь:** `/home/oracle/fitness-bot/`
**Бот:** `@stat_sila_bot`
**Python:** 3.12 + aiogram 3 + aiosqlite
**AI:** Claude API (claude-sonnet-4-6) — текст, анализ, КБЖУ — `services/ai_service.py`
**STT:** Groq Whisper (whisper-large-v3-turbo) — только транскрипция голоса
**БД:** SQLite (`fitness.db`)

## Запуск

Бот управляется **systemd** (`fitness-bot.service`). НИКОГДА не запускай через `nohup` — будет два процесса и двойные напоминания.

```bash
# Статус
sudo systemctl status fitness-bot

# Перезапустить (после изменений кода)
sudo systemctl restart fitness-bot

# Логи
tail -f /home/oracle/fitness-bot/bot.log

# Проверить что один процесс
pgrep -fa "fitness-bot/venv"
```

```

│                                 имя берётся из Telegram-профиля автоматически)
│   ├── workout.py              — тренировки, логирование подходов, таймер отдыха
│   ├── nutrition.py            — питание, голосовой ввод еды, КБЖУ
│   ├── stats.py                — статистика (тоннаж, питание)
│   ├── settings.py             — настройки пользователя, обновление веса
│   └── nav.py                  — хелпер send_nav (единый стиль навигации)
├── keyboards/keyboards.py      — все клавиатуры (main_menu, workout_menu, nutrition_menu и др.)
├── services/
│   ├── ai_service.py           — Groq: parse_food, analyze_workout, get_exercise_technique, transcribe_voice
│   └── scheduler.py            — APScheduler: напоминания о приёмах пищи
├── states/states.py            — FSM состояния (WorkoutLogging, FoodLogging, Setup)
└── venv/                       — виртуальное окружение
```

## База данных — таблицы

| Таблица | Назначение |
|---------|-----------|
| `users` | профиль, цели по КБЖУ, текущая неделя/день программы |
| `user_program` | упражнения по (user_id, week_type, day_type) |
| `workouts` | сессии тренировок, тоннаж, RPE, is_finished, ex_index/set_index |
| `workout_sets` | отдельные подходы (вес, повторы, RPE, заметки) |
| `food_log` | приёмы пищи по датам (КБЖУ) |

## Что реализовано

- [x] Онбординг — 8 шагов FSM: возраст → вес → рост → цель → опыт → дни → оборудование → травмы/болезни
- [x] AI-генерация программы тренировок через Groq (для обычных пользователей)
- [x] Персональная hardcoded программа для Кирилла (из `program_data.py`, не через AI)
- [x] Логирование подходов с FSM (формат: `вес×повторения RPE-оценка`, например `80x5 RPE8`; RPE — шкала усилия 1–10, необязателен)
- [x] Таймер отдыха с обратным отсчётом и уведомлением за 10 сек
- [x] Возобновление незавершённой тренировки
- [x] AI-анализ тренировки после завершения (сравнение с прошлой)
- [x] Трекинг питания — текст или голос → Groq → КБЖУ
- [x] Напоминания о приёмах пищи (APScheduler)
- [x] Статистика: тоннаж, питание, прогресс
- [x] Настройки: вес командой `вес 75`, /start для перестройки программы
